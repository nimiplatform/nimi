# 模型管理

Nimi 在你的本地机器上运行 AI 模型。本指南介绍如何列出、下载和管理这些模型。

## 列出已安装的模型

查看当前系统上所有已安装的模型：

```bash
nimi model list
```

获取机器可读输出：

```bash
nimi model list --json
```

你也可以通过完整的环境诊断来查看已安装模型数量：

```bash
nimi doctor
```

## 下载模型

通过模型引用标识符下载指定模型：

```bash
nimi model pull --model-ref <ref>@latest
```

将 `<ref>` 替换为模型引用。`@latest` 标签表示下载最新可用版本。

## 生成时自动下载

当你执行本地生成而所需模型未安装时，Nimi 会以交互方式处理：

```bash
nimi run "What is Nimi?"
```

如果默认本地模型缺失，Nimi 会询问你是否要下载。确认后即可在同一命令中完成下载并继续生成。

要跳过确认提示并自动下载缺失的模型：

```bash
nimi run "What is Nimi?" --yes
```

`--yes` 参数告诉 Nimi 无需询问直接下载模型。

## 模型缺失时会发生什么

当生成请求所需的模型未安装时：

1. Nimi 检测到模型缺失
2. 未使用 `--yes`：Nimi 显示模型引用并询问是否确认下载
3. 使用 `--yes`：Nimi 立即开始下载
4. 下载完成后，生成正常进行

如果你拒绝下载提示，Nimi 会退出并显示错误信息，指明所需的模型。你可以随时通过 `nimi model pull` 手动下载。

## 模型存储位置

下载的模型存储在 Nimi 数据目录中。默认位置在 `~/.nimi/` 内。你可以通过以下命令验证配置和数据路径：

```bash
nimi doctor
```

输出中包含配置文件路径，该路径与下载的模型数据位于同一目录树中。
