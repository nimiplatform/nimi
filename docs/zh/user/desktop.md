# 桌面应用指南

Nimi Desktop 是一个基于 Tauri 构建的原生应用，提供 AI 交互、Mod 管理和运行时配置的图形界面。支持 macOS、Windows 和 Linux。

## 下载与安装

从 [Nimi 发布页面](https://github.com/nimiplatform/nimi/releases) 下载适用于你平台的最新版本。

- **macOS**：Apple Silicon 和 Intel 对应的更新归档（`.app.tar.gz`）
- **Windows**：NSIS 安装器（`.exe`）
- **Linux**：AppImage

在 macOS 上，先解压归档得到应用包，再将它移动到“应用程序”文件夹。Windows 上直接运行 NSIS 安装器。Linux 上为 AppImage 添加可执行权限后即可启动。

桌面端自动更新使用的稳定入口为 [https://install.nimi.xyz/desktop/latest.json](https://install.nimi.xyz/desktop/latest.json)。

## 首次启动

首次打开 Nimi Desktop 时：

1. 如果 Nimi 运行时尚未启动，应用会自动启动它
2. 你会看到主屏幕
3. 如果没有安装本地模型，应用会引导你下载一个
4. 你可以选择登录以连接 Nimi 平台，使用社交功能和云端同步

本地 AI 生成完全可以离线使用。登录是可选的，仅在使用联系人、世界和个人资料等平台功能时需要。

## 主界面

桌面界面由侧边栏和主内容区域组成。

### 侧边栏导航

左侧侧边栏提供所有主要功能的入口：

- **首页** -- 来自 Nimi 平台的动态、帖子和社交内容
- **联系人** -- 好友列表、好友请求和联系人详情
- **世界** -- 浏览 AI 世界和智能体并与之交互
- **探索** -- 发现新内容、世界和用户
- **Mod Hub** -- 浏览、安装和管理 Mod
- **运行时配置** -- Provider 设置、模型管理和运行时健康状态
- **设置** -- 账户、偏好、隐私、语言和高级选项

### 顶部栏

顶部栏显示当前页面标题和导航控件。在某些页面中还包含标签切换和操作按钮。

## AI 对话

### 开始对话

导航到某个世界或智能体即可开始 AI 对话。对话区域以时间线形式显示消息历史。在底部的输入框中输入消息，按 Enter 发送。

### 切换本地和云端模型

AI 路由通过"运行时配置"部分进行设置。你可以：

- 使用本地模型进行离线、私密的生成
- 切换到云端 Provider 以访问更大的模型
- 配置默认使用的 Provider 和模型

当前使用的模型和 Provider 会显示在运行时配置面板中。

### 消息时间线

对话以消息时间线形式展示你的消息和 AI 回复。时间线支持滚动浏览历史记录，并实时显示流式响应。

## Mod Hub

Mod 为 Nimi 扩展额外的功能、UI 面板和能力。

### 浏览 Mod

从侧边栏打开 **Mod Hub** 查看可用的 Mod。每个 Mod 显示名称、描述和当前状态。

### 安装 Mod

在 Hub 中选择一个 Mod 并安装。应用会下载并将 Mod 注册到运行时。

### 管理已安装的 Mod

在运行时配置中打开 **Mods** 面板查看所有已安装的 Mod。在这里你可以：

- 启用或禁用单个 Mod
- 查看 Mod 详情和设置
- 移除不再需要的 Mod

提供 UI 扩展的 Mod 会在应用中显示为额外的面板或路由。

## 运行时配置

运行时配置是本地 AI 设置的控制中心，分为以下几个页面：

### 概览

显示运行时健康状况、使用统计和当前配置的摘要。

### 本地引擎

配置本地 AI 引擎。查看引擎状态、调整本地模型设置和监控资源使用情况。

### 云端 Provider

添加、编辑和删除云端 Provider 配置。对于每个 Provider，你可以：

- 输入或更新 API key
- 测试连接性
- 将某个 Provider 设为云端路由的默认值

### 模型中心

浏览模型目录，查看已安装的模型，管理模型依赖项，以及下载新模型。模型中心显示：

- **已安装模型** -- 当前机器上的模型
- **目录** -- 可供下载的模型
- **依赖项** -- 每个模型所需的组件

### Mods

在运行时配置中查看和管理已安装的 Mod。

### 运行时健康

实时监控运行时进程状态、gRPC 连接性和本地引擎健康状态。

## 设置

### 账户

管理你的 Nimi 平台账户，登录或退出，查看账户详情。

### 偏好

自定义外观和行为，包括主题、显示密度和通知偏好。

### 语言与地区

设置应用语言和地区偏好。

### 隐私

查看和调整隐私设置及数据共享偏好。

### 安全

管理账户的安全设置。

### 数据管理

导出、导入或清除本地数据。

### 高级

访问开发者模式、调试工具和高级运行时配置选项。

## 键盘快捷键

| 快捷键 | 操作 |
|---|---|
| `Cmd/Ctrl + ,` | 打开设置 |
| `Cmd/Ctrl + Q` | 退出应用 |
| `Cmd/Ctrl + R` | 刷新当前视图 |
| `Cmd/Ctrl + Shift + I` | 切换开发者工具（需启用开发者模式） |
| `Enter` | 在对话中发送消息 |
| `Shift + Enter` | 在对话输入框中换行 |
| `Escape` | 关闭弹窗或面板 |

## 故障排除

### 应用无法启动

- 确认你的操作系统满足最低要求
- 在 macOS 上，检查应用是否在"系统设置"的"隐私与安全性"中被允许
- 尝试删除应用并重新安装

### 启动后白屏

- 重启应用
- 删除 Nimi Desktop 数据缓存目录后重新启动
- 检查操作系统层面的 GPU 或渲染问题，特别是在 Linux 上使用 Wayland 时

### 运行时无法连接

- 桌面应用会自动启动运行时，但如果启动失败，请在终端中手动运行 `nimi start`
- 在终端中运行 `nimi status` 验证守护进程是否正在运行
- 确认没有其他程序占用 46371 端口

### Mod 加载错误

- 在运行时配置中禁用有问题的 Mod 并重启应用
- 检查 Mod 是否与当前 Nimi 版本兼容
- 从 Mod Hub 重新安装该 Mod

### 其他问题

- 查看运行时日志：`nimi logs --tail 100`
- 在终端中运行 `nimi doctor` 进行完整诊断
- 更多解决方案请参阅[故障排除指南](troubleshooting.md)
