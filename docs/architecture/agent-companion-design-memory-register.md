# Agent Companion Design Memory Register

This register preserves long-lived design memory from the closed companion
substrate work in a tracked document. It is a reader and planning artifact.
Normative authority remains in `.nimi/spec/**` and admitted
`apps/**/spec/**`; closed lifecycle topics are evidence only.

This document does not reopen broad APML, a broad Event API, wildcard broker
semantics, SDK-owned app-event emission, or closed-topic implementation packets.

Detailed file/body-level preservation is tracked in
[Agent Companion Design Corpus Preservation](./agent-companion-design-corpus-preservation.md).
Use this register for the compact disposition matrix and the corpus
preservation page for retained source-body detail.

## Disposition Vocabulary

| Disposition | Meaning |
| --- | --- |
| `admitted` | Active authority exists in `.nimi/spec/**` or admitted `apps/**/spec/**`. |
| `implementation_open` | Product-required work is still open under an admitted authority. |
| `projection_only` | Semantics must travel through typed Runtime/SDK/App projections, not proposed surface syntax. |
| `deferred` | Future-capable design memory is retained but not required for the current continuation line. |
| `blocked` | Requires parent manager and active authority redesign before implementation. |
| `superseded` | Replaced by narrower or different current active authority. |
| `retired` | Intentionally outside this continuation line. |
